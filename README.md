# SNHU-CS-305
Artemis Financial Vulnerability Assessment Report

Reflection

Artemis Financial was a financial services company that needed to secure its web application used for handling sensitive client data. The main issue was identifying and fixing security vulnerabilities in their software, especially related to data protection, encryption, and secure communication. The company wanted to ensure that customer financial information remained confidential and protected from potential cyber threats.

One thing I did well during the vulnerability assessment was systematically identifying risks using both manual review and automated tools. I focused on locating weak points such as outdated dependencies, insecure communication protocols, and missing encryption layers. Coding securely is critical because it prevents data breaches, protects user privacy, and maintains trust. Strong software security adds value by reducing financial risk, avoiding legal issues, and improving a company’s reputation.

The most challenging part of the assessment was interpreting the vulnerability scan results and deciding which issues were the highest priority to fix. However, this was also the most helpful part because it taught me how to think like a security analyst and prioritize real-world risks instead of just fixing everything blindly.

I increased layers of security by implementing encryption (HTTPS), improving dependency management, and addressing known vulnerabilities identified by tools like OWASP Dependency-Check. In the future, I would continue using tools such as static code analysis, dependency scanners, and penetration testing methods to assess vulnerabilities and choose the best mitigation strategies.

To ensure the application remained functional and secure, I tested the software after making changes and reviewed the code to confirm that fixes did not break existing functionality. After refactoring, I reran vulnerability scans and verified that no new issues were introduced, ensuring both stability and security.

Some key resources and tools I used include OWASP Dependency-Check, secure coding practices, and vulnerability assessment frameworks. These tools and practices will be useful in future work, especially in cybersecurity roles where identifying and mitigating risks is essential.

From this assignment, I can show future employers my ability to perform a full vulnerability assessment, apply secure coding practices, and use industry tools to improve software security. This demonstrates practical, real-world skills in cybersecurity and secure software development.
